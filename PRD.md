# ExpoPilot OS PRD
版本：V1.0
类型：MVP

## 1. 产品概述

### 产品名称
ExpoPilot OS

### 产品定义
ExpoPilot OS 是面向会务执行公司的现场运营 Agent 系统。
它负责把现场信号转成运营事件，把事件转成任务，把任务执行结果沉淀成项目复盘。

### 当前阶段目标
在现有前端原型基础上，完成一个最小可演示 MVP，而不是重建完整平台。

### MVP 主链路
事件识别 -> 任务下发 -> 执行反馈 -> 项目复盘

## 2. 用户与角色

### 使用方
- 会务执行公司

### 演示角色
- 项目经理
- 执行人员 1
- 展台接待 1

## 3. MVP 范围

### 后台端
- 项目列表
- 项目创建
- 区域配置
- 实时态势页
- 事件中心页
- 任务调度页
- 复盘分析页

### 工作人员端
- 当前任务
- 任务详情
- 状态反馈
- 历史任务

### 事件类型
- `entrance_congestion`

### 任务类型
- 补位
- 支援接待
- 导流
- 待命

### 任务状态流
- `created`
- `received`
- `processing`
- `completed`
- `exception`

## 4. 固定演示场景

### 项目
春季消费展

### 区域
- 入口区
- 主通道区
- 展台 A
- 展台 B

### 演示闭环
1. 后台进入“春季消费展”
2. 入口区触发 `entrance_congestion`
3. 系统生成 1 个“补位”任务并下发给“执行人员 1”
4. 工作人员端先反馈 `processing`
5. 再反馈 `completed`
6. 复盘页展示：
   - 1 个入口拥堵事件
   - 1 个补位任务
   - 1 次完成反馈

## 5. 功能要求

| ID | 功能 | 描述 | 优先级 | 验收 |
|---|---|---|---|---|
| F01 | 项目列表 | 展示项目入口 | Must | 可看到“春季消费展” |
| F02 | 项目创建 | 创建新项目 | Must | 可创建项目并进入配置 |
| F03 | 区域配置 | 配置四个演示区域 | Must | 四个区域可编辑并保存 |
| F04 | Mock 事件接入 | 接收 mock 事件或手工注入事件 | Must | 可生成 `entrance_congestion` |
| F05 | 事件中心 | 展示事件并可进入调度 | Must | 事件可进入待处理列表 |
| F06 | 任务调度 | 从事件生成任务并展示调度结果 | Must | 可为执行人员 1 生成“补位”任务 |
| F07 | 工作人员反馈 | 工作人员更新任务状态 | Must | 可回传 `processing` 和 `completed` |
| F08 | 复盘分析 | 汇总事件、任务和反馈结果 | Must | 复盘页按真实数据展示统计结果 |

## 7. 非目标
- 报名注册系统
- CRM
- lead capture
- 完整展商系统
- 复杂 CV 模型训练
- 深度可穿戴设备接入

## 8. 技术实施约束
- 必须基于当前现有仓库继续开发
- 当前仓库真实技术栈为：Vite + React + TypeScript
- 不迁移到 Next.js
- 不改造成 monorepo
- 不引入 FastAPI 或 PostgreSQL 作为当前 MVP 的前提
- 优先复用：
  - `src/App.tsx`
  - `src/domain/*`
  - `src/services/*`
  - `src/selectors/*`
  - `src/components/*`

## 9. 页面规则
- 主导航只保留当前 MVP 页面
- 非 MVP 页面从主导航移除
- 非 MVP 页面可保留路由，但统一显示“后续版本预留”
- 非 MVP 页面不新增复杂交互逻辑

## 10. 验收标准
MVP 完成仅以这条链路为准：
1. 事件被识别
2. 任务被下发
3. 执行状态被回传
4. 项目复盘自动生成
