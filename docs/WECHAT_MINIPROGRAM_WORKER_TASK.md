# 场脉工作人员小程序任务端

## 定位

这个小程序目录是离线演示版本，用来验证“工作人员扫码后看到的任务执行端”是否适合迁移到微信小程序。

它不替换现有 Web `/mobile`，也不接真实后端、真实登录、微信订阅消息或生产任务同步。

## 目录

```text
miniapp/worker-task-demo/
├─ app.js
├─ app.json
├─ app.wxss
├─ project.config.json
├─ sitemap.json
└─ pages/task/
   ├─ task.js
   ├─ task.json
   ├─ task.wxml
   └─ task.wxss
```

## 演示主线

| 项目 | 内容 |
|---|---|
| 主案例 | 入口 A 人流拥堵异常处置 |
| 使用角色 | 入口引导员 A |
| 派发来源 | 项目经理确认 / DispatchAgent 建议 |
| 任务位置 | A1 主入口 / 展馆入口 A |
| 到达要求 | 5 分钟内 |
| 反馈结果 | 完成分流、排队长度下降、继续观察 |

## 打开方式

1. 打开微信开发者工具。
2. 选择“导入项目”。
3. 项目目录选择：

```text
D:\code\expopilot-demo\miniapp\worker-task-demo
```

4. AppID 可以使用测试号或游客模式。
5. 编译后进入 `pages/task/task`。

## 当前边界

| 不做 | 原因 |
|---|---|
| 不接真实微信登录 | 当前只验证工作人员任务端体验 |
| 不接真实后端 | 避免影响 Web demo 主线 |
| 不接真实推送 | 订阅消息需要正式小程序主体和模板 |
| 不做跨设备同步 | 当前是离线演示，不承诺生产状态一致 |
| 不替换 `/mobile` | Web H5 仍是现有稳定扫码入口 |

## 后续生产化事项

| 事项 | 说明 |
|---|---|
| 小程序主体 | 需要注册并认证微信小程序 |
