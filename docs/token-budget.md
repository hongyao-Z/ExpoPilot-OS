# ExpoPilot OS Token Budget

## 当前策略

- 界面走查默认不走真实大模型，优先用确定性模拟数据保证稳定性。
- 后续智能体接入采用按需触发，不对每个事件自动调用。
- 事件解释、派发建议、复盘生成都保留预算位，但当前主要用于边界设计和命令行契约。

## 预算估算

- 单次事件解释：约 `2.05k`
- 单次派发建议：约 `2.58k`
- 单次复盘生成：约 `9.2k`
- 单项目单场试点上限：约 `71.6k`

## 控制原则

- 只取最近 8 分钟上下文
- 每次只带 top 5 相关事件
- 解释按需展开
- 复盘总结只在阶段结束时生成一次

## 当前结论

- ExpoPilot OS 当前版本可以在没有真实大模型调用的情况下完成界面走查和产品链路验证。
- 接真实智能体前，应先确定服务端代理、权限控制和审计策略。
