# QR Poster Guide

## 二维码指向

本地开发演示：

```text
http://localhost:5173/#/mobile
```

部署后：

```text
https://你的部署地址/#/mobile
```

`/#/mobile` 是公开 H5 展演页，不要求登录，不进入桌面端 AppFrame。

## 现场讲解口径

- 这不是完整移动后台，而是路演用轻量 H5 展演入口。
- 页面展示当前系统核心链路：现场信号 -> 事件识别 -> Agent 解释 -> 人工确认 -> 审计复盘。
- OpenClaw Adapter 只作为解释来源，不是执行器，不绕过本地风控与审计。

## 生成二维码

任意二维码生成工具均可。输入部署后的完整 URL：

```text
https://你的部署地址/#/mobile
```

如果只做本机演示，可以临时使用：

```text
http://localhost:5173/#/mobile
```

但注意：手机扫描 `localhost` 时会访问手机自己，不会访问电脑。因此现场扫码建议使用公网部署或局域网可访问地址。

## 手机打不开时检查

1. 是否部署到公网。
2. 是否使用了电脑的局域网 IP，而不是 `localhost`。
3. Vite 是否用 `--host 0.0.0.0` 暴露到局域网。
4. 手机和电脑是否在同一 Wi-Fi。
5. 防火墙是否拦截 `5173`。
6. 是否需要 ngrok / Vercel / Netlify 等临时部署。

## 本地局域网示例

电脑 IP 假设为 `192.168.1.20`，启动时使用：

```powershell
npm run dev -- --host 0.0.0.0
```

二维码地址改为：

```text
http://192.168.1.20:5173/#/mobile
```
