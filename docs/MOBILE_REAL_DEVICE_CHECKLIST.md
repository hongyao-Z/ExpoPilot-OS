# Mobile H5 真机检查

## 检查入口

正式部署后：

```text
https://expopilot.cn/#/mobile
```

本地局域网临时演示：

```powershell
npm run dev -- --host 0.0.0.0
```

然后用电脑局域网 IP：

```text
http://<电脑局域网IP>:5173/#/mobile
```

## 真机检查表

| 检查项 | 通过标准 |
|---|---|
| 首屏任务 | 能看到“入口 A 人流拥堵引导” |
| 位置 | 能看到 A1 主入口 / 展馆入口 A |
| 时限 | 能看到 5 分钟内到达 |
| 优先级 | 能看到高优先级 |
| 派发来源 | 能看到项目经理确认 / DispatchAgent 建议 |
| 联系人 | 能看到现场项目经理 |
| 操作按钮 | 确认接收、我已到达、开始处理、完成反馈、请求支援、现场异常均可见 |
| 触控尺寸 | 按钮手指可点，不需要放大页面 |
| 状态变化 | 点击按钮后状态和时间线同步变化 |
| 反馈区 | 能填写或展示现场备注 |
| 横向溢出 | 不出现横向滚动 |
| 对比度 | 室内展会灯光下文字可读 |

## 推荐测试设备

| 设备 | 用途 |
|---|---|
| iPhone / Safari | 检查 iOS 扫码体验 |
| Android / Chrome | 检查安卓默认体验 |
| 微信内置浏览器 | 检查海报扫码常见入口 |

## 常见问题

| 问题 | 处理 |
|---|---|
| 手机扫 localhost 打不开 | localhost 指向手机自己，必须用公网域名或电脑局域网 IP |
| 页面很慢 | 优先使用国内 HTTPS 域名，不用 GitHub Pages 作为主链路 |
| 页面显示旧版本 | 清理浏览器缓存，或在二维码 URL 后加版本参数 |

示例：

```text
https://expopilot.cn/#/mobile?v=demo-202605
```
